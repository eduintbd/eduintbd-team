Session Summary — 2026-05-19
================================

BUGS FIXED
----------

1. Facebook OAuth connect failing for non-connected channels (e.g. Sunshine's collection)
   - Root cause: FACEBOOK_APP_ID, FACEBOOK_APP_SECRET, FACEBOOK_OAUTH_STATE_SECRET,
     and DISPATCHER_SECRET were not set in Supabase edge function secrets.
   - Fix: set all four secrets via Supabase dashboard / CLI.
   - Functions: facebook-oauth-start, facebook-oauth-callback, dispatch-scheduled-posts

2. Email — sharraf's inbox failing with generic "non-2xx" error
   - Root cause: wrong Purelymail password stored in employees table.
   - Fix: reset via User Management → sharraf → Edit Email → re-enter correct password.
   - Also improved emailProxy error handling to surface real server error instead of generic message.
   - File: src/pages/Email.tsx

3. Email — mahedi's sender showing "unknown"
   - Root cause: IMAP ENVELOPE FROM regex only matched quoted display names;
     failed silently when name was NIL (no display name).
   - Fix: updated regex to handle both ("Name" NIL "user" "host") and (NIL NIL "user" "host").
   - File: supabase/functions/purelymail-proxy/index.ts

4. Email — mahedi's email body always empty
   - Root cause: SimpleImapClient.sendCommand only collected lines starting with "*";
     IMAP literal body content (sent after a {N} size marker) was silently discarded.
   - Fix: added readLiteral(size) method; sendCommand now detects {N} at end of line
     and reads the following N chars as literal data.
   - File: supabase/functions/purelymail-proxy/index.ts

5. Email — get_thread crashing after attachment support added
   - Root cause: used Promise.all for three sequential IMAP commands on the same
     connection — commands interleaved and corrupted each other's responses.
   - Fix: run fetchHeaders, fetchBody, fetchBodyStructure sequentially (await each).
   - File: supabase/functions/purelymail-proxy/index.ts
